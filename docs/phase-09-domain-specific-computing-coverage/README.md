## Phase Contract

Domain coverage documents are proof obligations for the whole-stack thesis. Each domain names the incumbent languages or platforms Gravity aims to replace, the profiles and backends required, the interop boundaries that remain, and the concrete examples that prove credible coverage.

The phase is successful when each domain has at least one end-to-end slice that compiles through ordinary Gravity syntax, profiles, effects, capabilities, safety checks, artifacts, and conformance evidence.

## Shared Evidence

- Each domain identifies required profiles, backends, runtime services, schemas, capabilities, and safety/proof artifacts.
- Replacement claims are scoped to explicit slices instead of broad marketing claims.
- Domain examples include both accepted programs and rejected programs that demonstrate Gravity-specific safety and capability rules.
